## 2026 Blackwell 推理架构：无感随处启动技术手册

### 1. 核心目标 (Objective)

构建一套**不可变 (Immutable)** 的底层基础设施与**持久化 (Persistent)** 的全局 Python 运行环境。

- **确定性：** 保证 CUDA、GCC、OpenSSL 等底层二进制库版本在不同目录下完全一致。
- **无感性：** 用户无需关心路径切换，执行 `nix develop` 或 `nix run` 即可自动挂载所有加速引擎。
- **硬件压榨：** 针对 Blackwell (RTX 5090) 的 NVFP4 硬件特性进行编译器级别的路径优化。

### 2. 技术栈选择 (Tech Stack)

- **外层锚点 (Nix Flakes):** 锁定 `nixpkgs` (unstable 分支)，管理 CUDA Toolkit 12.8、C++ 编译器及 Linux 动态链接库。
- **环境桥接 (Micromamba):** 替代传统 Conda，提供更快的依赖解析速度，并将其 Root 设为全局绝对路径。
- **推理引擎 (vLLM):** 开启 Blackwell 架构特定的 FP8/NVFP4 算子支持。
- **模型层 (GLM-4.7-Flash-NVFP4):** 利用硬件原生 4-bit 浮点单元，实现低延迟、高吞吐。

### 4. 终极 `flake.nix` 配置 (全局路径版)

将此文件保存在你的配置目录（如 `~/infra/vllm-5090/`）中。

```nix
{
  description = "Global-Scoped RTX 5090 Blackwell AI Infrastructure";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    flake-utils.url = "github:numtide/flake-utils";
  };

  outputs = { self, nixpkgs, flake-utils }:
    flake-utils.lib.eachDefaultSystem (system:
      let
        pkgs = import nixpkgs {
          inherit system;
          config.allowUnfree = true;
        };

        # 路径常量：全局绝对路径
        GLOBAL_MAMBA_ROOT = "$HOME/.cache/nix-vllm-mamba";
        ENV_NAME = "glm4-env";

        runtimeLibs = with pkgs; [
          stdenv.cc.cc.lib
          linuxPackages.nvidia_x11
          cudaPackages_12.cuda_cudart
          cudaPackages_12.libcublas
          zlib
          glibc
        ];
      in
      {
        devShells.vllm-5090 = pkgs.mkShell {
          name = "vllm-5090-global-shell";

          buildInputs = with pkgs; [
            cudaPackages_12.cuda_nvcc
            micromamba
            git
            pkg-config
          ];

          shellHook = ''
            # 1. 注入动态库路径 (Nix Store + WSL 驱动路径)
            export LD_LIBRARY_PATH="${pkgs.lib.makeLibraryPath runtimeLibs}:/usr/lib/wsl/lib:$LD_LIBRARY_PATH"

            # 2. 强制指定全局 Mamba 根路径
            export MAMBA_ROOT_PREFIX="${GLOBAL_MAMBA_ROOT}"
            eval "$(micromamba shell hook -s bash)"

            # 3. 自动检测并初始化环境
            if [ ! -d "${GLOBAL_MAMBA_ROOT}/envs/${ENV_NAME}" ]; then
              echo "🛠️ 正在全局路径初始化推理环境..."
              micromamba create -n ${ENV_NAME} python=3.12 -y
              micromamba run -n ${ENV_NAME} pip install --pre torch --index-url https://download.pytorch.org/whl/nightly/cu128
              micromamba run -n ${ENV_NAME} pip install vllm>=0.14.0 flashinfer
            fi

            micromamba activate ${ENV_NAME}
            echo "✅ 环境已就绪！当前路径: $(pwd)"
            echo "🚀 算力核心: RTX 5090 (Blackwell) | 模型: GLM-4.7-Flash-NVFP4"
          '';
        };

        # 注册全局运行命令
        apps.serve = {
          type = "app";
          program = "${pkgs.writeShellScript "serve-glm4" ''
            export LD_LIBRARY_PATH="${pkgs.lib.makeLibraryPath runtimeLibs}:/usr/lib/wsl/lib:$LD_LIBRARY_PATH"
            export MAMBA_ROOT_PREFIX="${GLOBAL_MAMBA_ROOT}"
            eval "$(micromamba shell hook -s bash)"
            micromamba activate ${ENV_NAME}
            python -m vllm.entrypoints.openai.api_server --model GadflyII/GLM-4.7-Flash-NVFP4 --trust-remote-code
          ''}";
        };
      });
}

```

### 5. 边界与约束 (Boundaries & Constraints)

- **硬件绑定：** \* 此配置强依赖 NVIDIA 590.xx+ 驱动。
- 仅限支持 WSL2 的 Windows 11 环境。

- **存储空间：** \* 全局 Mamba 路径位于 `$HOME/.cache/`，建议保留至少 20GB 空间（包含显存优化库和模型分片）。
- **版本约束：** \* Python 版本限制为 3.12。
- CUDA Toolkit 版本强制锁定为 12.8 以匹配 Blackwell 初始优化周期。

### 6. 注意事项 (Precautions)

1. **Registry 注册：** 为了实现“任意地方启动”，必须在安装完 flake 后运行一次：
   `nix registry add vllm-infra /home/你的用户名/path-to-flake-dir`
   此后，在任何地方输入 `nix develop vllm-infra#vllm-5090` 即可瞬间激活。
2. **显存争抢：** RTX 5090 拥有 32GB 显存，但在 WSL 环境下，由于 DWM (Windows 桌面管理器) 的开销，可用显存可能略低于 32GB。若出现 OOM，请将 `--gpu-memory-utilization` 下调至 `0.9`。
3. **Blackwell 编译冷启动：** 首次运行 vLLM 时，系统会编译 Blackwell 特定的 CUDA Kernel，这可能需要 2-3 分钟，期间 CPU 占用会激增，属正常现象。
